We present SC2Arena, a comprehensive benchmark for evaluating LLMs in StarCraft II, and SC2Agent, a hierarchical agent framework that achieves state-of-the-art performance through self-correction and self-improvement.

Abstract

Evaluating large language models (LLMs) in complex decision-making is essential for advancing AI's ability to plan strategically and adapt in real-time. However, existing benchmarks for tasks like StarCraft II fail to capture the game's full complexity, including complete game context, action spaces, and race configurations. To address this, we present SC2Arena, a benchmark that fully supports all playable races, low-level action spaces, and optimized text observations to handle spatial reasoning challenges. We also introduce SC2Agent, a hierarchical framework that integrates strategic planning and tactical execution with iterative self-correction and continuous improvement through fine-tuning on high-quality gameplay data. Key components include a Planner-Executor-Verifier structure and an RL-inspired scoring system for selecting training samples. Experiments demonstrate SC2Agent’s ability to achieve state-of-the-art performance, with smaller models defeating strong opponents and larger models excelling against advanced AI, showcasing its potential for broader AI research.

SC2Arena Benchmark

We introduce SC2Arena, a structured, text-based benchmark designed to evaluate LLM agents in StarCraft II. It provides standardized observation and action interfaces, preserving the game's original strategic depth and complexity.

Overview of StarCraft II and its Races

StarCraft II is a highly complex real-time strategy game demanding proficiency in macro-management, micro-control, and strategic decision-making under partial information. Each of its three races offers a unique playstyle:

  • Terran: A versatile human race with mechanized, repairable units, known for powerful Bio-ball micro-tactics.
  • Zerg: A swarm-based alien race that overwhelms opponents with rapidly breeding biological units, leveraging the Creep Spread mechanism for map control.
  • Protoss: An advanced alien race with powerful, shielded units that can be warped into battle instantly via their Pylon network.

Key Features and Improvements

Unlike prior benchmarks that often focus on isolated micro-management or high-level strategy, SC2Arena offers a comprehensive evaluation platform. Key improvements include:

  • Full-Length Game Context: Supports complete games, combining long-term strategic planning with real-time tactical execution.
  • All Playable Races: Full support for Terran, Zerg, and Protoss, enabling evaluation across diverse strategic matchups.
  • Complete Low-Level Action Space: Provides precise control over all in-game actions, preserving the game's original complexity.
  • Agent-vs-Agent Gameplay: Crucially enables direct competition between agents, facilitating robust performance assessment and ranking via an Elo system.

Optimized Observation Space for LLMs

To enable LLMs to play effectively, raw game states must be converted into textual observations. This presents challenges like spatial reasoning difficulty and information overload. SC2Arena addresses these with two key techniques:

  • Proximity-based Unit Ordering: We enhance spatial reasoning by ordering units based on their proximity to each other, starting from the main base. This greedy nearest-neighbor approach helps the LLM infer spatial relationships and identify interactable structures (e.g., a Barracks with an attached Tech Lab).
  • Unit Aggregation: We reduce information overload by aggregating routine units, like workers, into a single representation. This preserves core information (e.g., total number of workers, their tasks) while removing redundant details that can distract the agent from strategic decision-making.

These optimizations are proven to significantly improve agent performance by making the textual observations more concise and spatially aware.

SC2Agent Framework

We introduce SC2Agent, a closed-loop, self-improving, and self-correcting LLM agent framework for StarCraft II. Its design allows even smaller LLMs to achieve competitive performance against challenging opponents by continuously learning from gameplay.

Hierarchical Self-Correction Framework

To tackle the vast action space and strict syntax requirements of StarCraft II, SC2Agent employs a two-tier architecture with integrated self-correction:

  • Planner: A high-level LLM module that generates strategic commands in natural language (e.g., "Build a Barracks") based on game observations.
  • Executor: A low-level LLM module that translates the Planner's commands into precise, executable JSON actions.
  • Verifier: Both the Planner and Executor are equipped with Verifier modules. These LLM-based verifiers check the outputs against predefined rules (e.g., resource constraints, unit validity, JSON syntax). If an output is invalid, the Verifier provides feedback, prompting the Planner or Executor to iteratively refine its output until it is correct and feasible.

Self-Improvement via Fine-tuning

A key feature of SC2Agent is its ability to learn and improve over time. This is achieved through a self-improvement loop based on Supervised Fine-Tuning (SFT):

  • Data Collection: The agent collects data from its own winning game trajectories. Each data point consists of an observation, a high-level command, and the resulting low-level action.
  • RL-Inspired Scoring: To ensure data quality, we use a scoring function inspired by reinforcement learning. It evaluates each action based on its discounted future impact on key game metrics (e.g., army value, economic output). Only actions with scores above a certain threshold are selected for training.
  • Fine-tuning: The high-quality data is used to fine-tune the Planner, Executor, and Verifier models. This process distills strategic expertise from successful gameplay directly into the agent's models, enabling continuous performance improvement and allowing smaller models to defeat strong opponents.
